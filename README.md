# AstrBot 天使之笑

允许 LLM 在回复中插入表情包，并支持通过工具收入新表情。

## 工作原理

1. 插件扫描 `memes/` 目录，生成可用 meme 列表注入到 LLM 提示词中。
2. LLM 在回复中使用 `:meme名:` 格式引用表情。
3. 发送前，插件将 `:meme名:` 替换为对应图片。
4. LLM 看到适合做表情的图片时，可调用 `meme` 工具收入库存。

## 目录结构

插件数据目录：`data/plugin_data/astrbot_plugin_angel_smile`

```
memes/
├── 坏笑.webp              # 单图 meme
├── 无语/                   # 多变体 meme
│   ├── 无语.webp
│   ├── 无语(2).webp
│   └── 无语(3).webp
└── image_dhash_index.json  # 图哈希去重库（不参与 meme 列表）
```

### 规则

- 根目录图片文件：文件名（不含扩展名）即为 meme 名。
- 第一层文件夹：文件夹名即为 meme 名，内部图片为变体。
- 文件夹优先于同名根目录文件。
- 非图片文件、深层目录、以 `.` 开头的名称均被忽略。

## LLM 工具

工具名：`meme`

参数：

| 参数 | 必填 | 说明 |
|------|------|------|
| `emotion` | 是 | 贴纸名，即聊天中使用的 `:贴纸名:` |
| `path` | 是 | 图片引用：本地路径、`file:///` 或 `http(s)` URL |

### 入库行为

- 所有图片统一转换为 WebP 格式保存。
- 首次入库：`memes/<emotion>.webp`
- 第二次同名入库：升级为文件夹 `memes/<emotion>/`，原图移入，新图编号 `(2)`。
- 后续同名入库：递增编号 `(3)`, `(4)`, ...

### 去重

- 入库前通过 dHash 检测近似重复图片，命中则拒绝保存。
- 已删除图片的 hash 仍保留在索引中，防止同图被再次收入。

## 使用方式

LLM 在回复中直接写 `:meme名:`，例如：

```
好的，我来看看 :坏笑:
```

插件会将 `:坏笑:` 替换为对应图片发送。

## 配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `max_stickers_per_message` | 1 | 每条消息最多使用的贴纸数量 |
